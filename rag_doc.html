How this simulator works

Files are parsed entirely client-side: PDFs via pdf.js, Word docs via mammoth.js, and images via Tesseract.js OCR — nothing is uploaded anywhere. Text is embedded using weighted word-frequency vectors (a simplified stand-in for a real neural embedding model), so results are driven by real keyword/semantic overlap between your query and documents. Similarity is true cosine similarity; the scatter plot is a random 2D projection of that same vector, just for visualization. OCR and multi-page PDF parsing can take a few seconds per file.

Built as an educational RAG simulator — all computation runs locally in your browser.
