Overview

This is the API surface a production version of this simulator would expose on a CPaaS platform. Base URL and auth follow common CPaaS conventions (Bearer key, per-enterprise scoping, JSON everywhere).

# Base URL https://api.yourcpaas.com/sms-rag # Auth — every request Authorization: Bearer YOUR_API_KEY Content-Type: application/json X-Enterprise-Id: ent_8k2j9x

▶ This one actually runs

A static HTML page can't host a network endpoint by itself, so alongside this simulator you also got server.js + package.json + .env.example — a real Node/Express server implementing every endpoint documented on this page, including live pass-through to Claude and Gemini.

Runs entirely offline in simulated mode (no keys needed). Pass provider: "claude" or "gemini" plus an api_key to route generation through a live model instead — the same 160/70-char validator still runs on whatever comes back.

POST/v1/rag/sms/generate

Runs the full RAG pipeline: embeds the query, retrieves top-K templates from the enterprise's vector store, assembles context, generates output (locally or via a live LLM), and enforces GSM-7/Unicode length limits.

GSM-7 extendedcounts 2 septets/char—^ {{ }} \ [ ] ~ | € and similar
UCS-2 (Unicode)70 chars67 charsHindi, Arabic, CJK, emoji, or any non-GSM-7 char

One non-GSM-7 character anywhere in the message forces the entire message to UCS-2 — this simulator enforces that rule exactly, same as real carriers.

This simulator's "Run Pipeline" button executes the same five stages client-side with a lightweight local embedding model, standing in for the embedding + vector-DB + LLM calls a real deployment would make.

Scaling a high-throughput SMS engine

— Stateless retrieval/generation workers behind a load balancer, autoscaled on queue depth.
— Vector store sharded per enterprise namespace (e.g. pgvector, Pinecone, Weaviate) to isolate tenants and keep ANN search sub-20ms at scale.
— Redis cache for repeat/near-duplicate queries (common in campaign blasts) to cut LLM calls.
— Async queue (Kafka/SQS) absorbs bursty campaign-generation traffic ahead of the SMS send layer.
— Batched embedding + batched LLM inference for throughput; streaming not required since output is ≤160 chars.

Guardrails & compliance

— Hard length/encoding validator runs after every generation — never trust the model alone for the 160/70 rule.
— Language-match filter restricts retrieval to same-language chunks before ranking, not just after.
— DLT/regulatory template mapping (e.g. India TRAI) per matched template group.
— PII redaction on ingested templates; brand/tone allow-list per enterprise.
— Per-enterprise API keys, IP allow-listing, and full audit log of generated + sent messages.

Client-side simulation — embeddings, retrieval and generation all run locally in your browser for teaching purposes.
